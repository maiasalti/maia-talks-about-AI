*By Maia Salti*

When people say "AI," they usually don't mean some abstract field of study — they mean ChatGPT, Claude, Gemini. Large Language Models. These chatbots that somehow talk back.

The four big players right now: **OpenAI's GPT** (which powers ChatGPT), **Anthropic's Claude**, **Google's Gemini**, and **Meta's Llama**. They differ in training data, fine-tuning, parameter counts, and other tweaks — but underneath, they're built on the same core architecture.

So this post is about that core. How does an LLM actually work? I spent a while learning, and here's my attempt to explain it as simply as possible — with the math tucked away in optional dropdowns for anyone who wants to dig deeper.

## Tokenization

### Intuition

An LLM doesn't look at words and sentences the same way that we do.

Before the model does anything, the text gets chopped up into chunks (not necessarily words) called **tokens**.

E.g. *"I like international pineapple on pizza"* → `[I] [like] [inter] [national] [pine] [apple] [on] [pizza]` → `[20] [435] [2341] [132] [487] [536] [32] [2847]`

![Tokenization with IDs](/images/blog-1/tokenization-example.jpg)

Each token gets converted to an ID (a unique number), meaning the sentence gets turned into a list of integers.

If you take away one thing it is this: **everything that the model knows about language, it learned by finding patterns in sequences of these numbers.**

### How the tokenizer works

The model starts by scanning a massive pile of text, and learning which chunks show up often. The standard method — **Byte Pair Encoding** — starts with individual characters, then repeatedly finds the most frequent pair of characters and turns that group into a single new token. It does this tens of thousands of times.

Frequent words keep getting promoted and eventually get their own single-token slot.

For example, start with 'h'. As it scans through text, it notices 'he' comes up very frequently, so 'he' becomes a single token. Then as it keeps scanning, it finds that 'the' comes up frequently, so that becomes a single token. Whereas if you have a token like 'inter' and 'international' doesn't come up often enough in the text, 'inter' will stay as a single token and 'national' may become a separate token.

![BPE merging the word "lower"](/images/blog-1/tokenization-bpe.jpg)

## Embeddings

I would say embedding is the hardest concept to fully understand in depth, so I will try my best to explain it as simply as possible. For simplicity I will replace 'token' with 'word' as it's easier to understand that way.

After tokenization, each word essentially gets given a set of coordinates that place it on a map (picture it like a web of words). Words that have similar meaning — e.g. King and Queen — would sit close together.

However, your typical graph (two-dimensional — horizontal x value, vertical y value) isn't sufficient for mapping words. There are many factors defining the "meaning" of a word. It's like trying to fit all the words in the dictionary into two different categories (e.g. positive and negative): it doesn't really describe the words very well to categorize them that way.

Real embedding coordinates can be **hundreds or thousands of dimensions** to account for the complexity of the English vocabulary and how it's used. (It really gives you a better appreciation of what our brains are capable of.) The higher dimensions allow for separation of the different semantic meanings and contexts.

![A simplified 2D embedding map](/images/blog-1/embedding-map.jpg)

That is essentially embedding. **It gives every token a list of numbers (a vector) that actually gives it meaning.** The model learns the position of each word during training.

<OptionalMath>

An embedding is a lookup into a big table, of dimension (number of tokens) × (number of dimensions on the describing vector).

Then use a 1 × |V| matrix with 0s everywhere and a 1 in the nth column (also called a **one-hot vector**) to get the word with ID number n (ID is the unique number assigned to each word).

Multiply it by the embedding matrix to retrieve the vector corresponding to the word.

![Embedding matrix retrieval](/images/blog-1/embedding-matrix.jpg)

## Attention

Now each word has a meaning assigned to a vector. But the issue is, a word can have different meanings depending on context. For example, "bank" by a river and "bank" with money have the same vector, but have different meanings.

**Attention fixes this by letting every word look around at all the other words in the sentence and decide which ones are relevant to it**, and pulls in a blend of their information to update its own vector.

Each word essentially asks: *"which other words here matter for understanding me?"*

<OptionalMath>

Each word's vector is multiplied by three different learned matrices to create 3 new vectors:

- **Query:** What am I looking for?
- **Key:** What am I about?
- **Value:** What content do I hand over if I'm selected?

![Producing Q, K, V vectors for token 3](/images/blog-1/attention-qkv.jpg)

Each word dot products its Query vector against every word's Key vector → creates a grid of relevance scores.

The scores are divided by √(dimension of each key vector) to scale them.

Then run the values through **softmax** so each word's scores become weights that sum to 1.

![Q·K dot products, scaled, then softmaxed](/images/blog-1/attention-scores.jpg)

Each word's new vector = the weighted sum of all the Value vectors, using its weights.

## Training

All of the training is based on this premise: **predict the next word.**

Grab a chunk of the text, hide the next word, let the model guess. It will output a probability for every possible next token. It will then guess the word it has calculated with the highest probability and then compare it to the word that actually came next.

It will then be rewarded if it's right and penalized if it's wrong. If it was *confidently* wrong it will be penalized more so than if it was only slightly confident (lower probability). This is called the **loss** — how wrong the model was.

We then adjust every parameter — every entry in the embedding table, and in the Query, Key, and Value matrices — to minimize this loss using **gradient descent**.

## Final Prediction

The vector of the last word in the sentence is multiplied by a final weighted matrix, which then outputs a probability for each word in the dataset. That probability represents the chance of that word being the next correct word.

E.g. *"The Sky Is …"* → `Blue = 0.86, Falling = 0.08, Red = 0.03, Not = 0.02, Here = 0.01`

![Final prediction example](/images/blog-1/final-prediction.jpg)

The model picks the most likely word, and that's its answer.

## So how do you define AI?

That's it. That's the whole machine. An LLM is a massive system of numbers that has learned, over billions of examples, to predict the next word in a sequence. Everything you see it do — answer questions, write essays, debug code — is downstream of that single objective.

Is that *intelligence*? That's a much harder question. I'll save it for another post.
